 ASTRACAT DNS-резолвер 🚀

✨ Новый релиз на Rust! ✨

Добро пожаловать в ASTRACAT — высокопроизводительный, асинхронный и рекурсивный DNS-резолвер, написанный на Rust. Этот проект создан для тех, кто ищет молниеносную скорость и надёжность при работе с DNS.
🌟 Преимущества

    ⚡️ Безумная скорость: Благодаря асинхронной архитектуре на базе Tokio и оптимизированному кэшу, ASTRACAT обрабатывает запросы невероятно быстро.

    🛡️ Надёжность и безопасность: Использование Rust обеспечивает безопасность памяти и защиту от распространённых ошибок, гарантируя стабильную работу сервера.

    🧠 Умное кэширование: Встроенный механизм кэширования и префетчинга позволяет резолверу обновлять записи до того, как они истекут, что значительно сокращает задержки.

    📈 Полностью рекурсивный: ASTRACAT самостоятельно находит DNS-записи, начиная от корневых серверов, что обеспечивает независимость и точность.

💻 Описание реализации

Резолвер ASTRACAT построен на асинхронной архитектуре, что позволяет ему одновременно обрабатывать множество клиентских запросов.

    Конкурентность: Основной цикл сервера в main() принимает UDP-пакеты. Каждый входящий запрос немедленно обрабатывается в отдельном асинхронном таске (tokio::spawn), что предотвращает блокировку и позволяет серверу мгновенно реагировать на новые запросы.

    Рекурсивный поиск: Функция recursive_lookup_with_cache выполняет итеративный поиск. Она отправляет запросы корневым серверам, а затем следует по ответам-ссылкам (delegation), переходя на серверы следующего уровня (TLD, авторитетные серверы), пока не найдет финальный ответ. Это гарантирует, что резолвер всегда находит актуальные данные.

    Кэш и префетчинг: Для хранения DNS-записей используется DashMap, потокобезопасная хэш-таблица. Это позволяет всем асинхронным таскам безопасно и без блокировок обращаться к кэшу. Фоновый поток постоянно проверяет кэш и, если время жизни записи (TTL) подходит к концу, инициирует префетчинг (обновление) записи заранее, чтобы избежать задержек для будущих запросов.

    Обработка ошибок: Для простого и надёжного управления ошибками используется библиотека anyhow, которая позволяет эффективно обрабатывать ошибки по всей цепочке вызовов.

⚙️ Как это работает (Технологии)

ASTRACAT использует передовые технологии для достижения своей производительности:

    Rust: Современный, безопасный и высокопроизводительный язык, который позволяет писать эффективный и надёжный код.

    Tokio: Асинхронный рантайм, который даёт возможность обрабатывать тысячи запросов одновременно, не блокируя основной поток.

    hickory-proto: Мощная библиотека для работы с DNS-протоколом, позволяющая легко отправлять и обрабатывать запросы.

    DashMap: Потокобезопасная, lock-free хэш-таблица, которая обеспечивает быстрый и одновременный доступ к кэшу без блокировок.

🚀 Как установить и запустить

    Клонируйте репозиторий:

    git clone https://github.com/ваш_репозиторий/The-ASTRACAT-DNS-Resolver.git
    cd The-ASTRACAT-DNS-Resolver

    Соберите проект в релиз-режиме:

    cargo build --release

    Запустите резолвер:

    RUST_LOG=info cargo run --release

🚀 Запуск в качестве демона (systemd)

Для того чтобы приложение работало в фоновом режиме и автоматически перезапускалось в случае сбоев, рекомендуется использовать systemd.

    Создайте файл службы:
    Создайте файл astracat-dns.service в директории /etc/systemd/system/ с помощью следующей команды:

    sudo nano /etc/systemd/system/astracat-dns.service

    Добавьте конфигурацию:
    Вставьте в файл следующий текст, заменив /path/to/your/dns-resolver-rust/ на фактический путь к вашему проекту.

    [Unit]
    Description=ASTRACAT DNS Resolver
    After=network.target

    [Service]
    ExecStart=/path/to/your/dns-resolver-rust/target/release/dns-resolver-rust
    WorkingDirectory=/path/to/your/dns-resolver-rust
    Restart=always
    RestartSec=3
    Environment=RUST_LOG=info

    [Install]
    WantedBy=multi-user.target

        ExecStart: Полный путь к вашему исполняемому файлу.

        WorkingDirectory: Рабочая директория проекта.

        Restart=always: Обеспечивает автоматический перезапуск при сбоях.

    Запустите службу:
    После сохранения файла выполните команды для перезагрузки systemd, включения и запуска службы:

    sudo systemctl daemon-reload
    sudo systemctl enable astracat-dns.service
    sudo systemctl start astracat-dns.service

    Проверка статуса:
    Чтобы убедиться, что служба успешно запущена, используйте:

    sudo systemctl status astracat-dns.service
