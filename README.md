# ASTRACAT DNS Resolver

ASTRACAT DNS Resolver - это рекурсивный DNS-резолвер, написанный на Go. Он предназначен для выполнения рекурсивных запросов DNS, начиная с корневых серверов, и разрешения доменных имен без использования внешних DNS-серверов (таких как Google Public DNS или Cloudflare DNS) для выполнения рекурсии.

## Ключевые преимущества

  * **Полная рекурсия с нуля:** ASTRACAT DNS Resolver выполняет полноценную рекурсию, начиная с корневых DNS-серверов. Это означает, что он не полагается на сторонние рекурсивные резолверы и может самостоятельно находить информацию о доменных именах.
  * **Устойчивость к отсутствию "glue records":** В отличие от некоторых простых резолверов, ASTRACAT DNS Resolver способен разрешать NS-серверы, для которых отсутствуют "glue records" (IP-адреса, предоставленные родительскими серверами). Он выполняет дополнительные запросы для получения IP-адресов таких NS-серверов, что повышает надежность разрешения.
  * **Написан на Go:** Использование языка Go обеспечивает высокую производительность, эффективность и простоту развертывания благодаря статически скомпилированным бинарным файлам.
  * **Легкость развертывания:** После компиляции резолвер представляет собой один исполняемый файл, который легко перемещать и запускать в различных средах Linux.
  * **Идеально подходит для изучения:** Простая и понятная кодовая база делает его отличным инструментом для тех, кто хочет глубже понять, как работает рекурсивный DNS.

## Начало работы

Эти инструкции помогут вам настроить и запустить ASTRACAT DNS Resolver на вашей системе Linux.

### Требования

  * **Go (Golang)**: Версия 1.16 или выше. Вы можете скачать Go с [официального сайта](https://golang.org/dl/).
  * **Git**: Для клонирования репозитория.

### Установка

1.  **Клонируйте репозиторий:**
    Откройте терминал и клонируйте репозиторий (предполагая, что он будет на GitHub):

    ```bash
    git clone https://github.com/ASTRACAT2022/The-ASTRACAT-DNS-Resolver.git
    cd The-ASTRACAT-DNS-Resolver
    ```

    

2.  **Установите зависимости:**
    Проект использует библиотеку `miekg/dns`. Установите ее:

    ```bash
    go mod tidy
    ```

    или

    ```bash
    go get github.com/miekg/dns
    ```

3.  **Соберите исполняемый файл:**
    Создайте исполняемый бинарник. Название `astracat-dns` будет использоваться в следующих шагах.

    ```bash
    go build -o astracat-dns .
    ```

### Запуск и тестирование (временный, для проверки)

Вы можете запустить резолвер вручную, чтобы убедиться, что он работает:

```bash
./astracat-dns google.com
```

Вы увидите логи процесса разрешения и в конце IP-адреса для `google.com`.

### Запуск как системный демон (рекомендуется для продакшна)

Для постоянной работы резолвера рекомендуется запускать его как системную службу с помощью `systemd`.

1.  **Создайте директорию для исполняемого файла:**
    Рекомендуется хранить исполняемые файлы сервисов в `/opt/` или `/usr/local/bin/`.

    ```bash
    sudo mkdir -p /opt/astracat-dns/
    sudo mv astracat-dns /opt/astracat-dns/
    ```

2.  **Установите права на выполнение:**
    Убедитесь, что исполняемый файл имеет права на выполнение.

    ```bash
    sudo chmod +x /opt/astracat-dns/astracat-dns
    ```

3.  **Опционально: Разрешите прослушивание порта 53 для непривилегированного пользователя:**
    Если вы планируете запускать сервис от имени непривилегированного пользователя (что очень рекомендуется для безопасности), вам нужно дать исполняемому файлу разрешение на прослушивание привилегированных портов (таких как 53).

    ```bash
    sudo setcap 'cap_net_bind_service=+ep' /opt/astracat-dns/astracat-dns
    ```

4.  **Создайте Systemd Unit-файл:**
    Создайте файл `/etc/systemd/system/astracat-dns.service`:

    ```bash
    sudo nano /etc/systemd/system/astracat-dns.service
    ```

    Вставьте следующее содержимое:

    ```ini
    [Unit]
    Description=The ASTRACAT DNS Resolver
    After=network.target

    [Service]
    Type=simple
    # User=astracat  # РЕКОМЕНДУЕТСЯ: Создайте пользователя: sudo useradd -r astracat
    # Group=astracat # РЕКОМЕНДУЕТСЯ: Укажите группу для созданного пользователя

    ExecStart=/opt/astracat-dns/astracat-dns
    WorkingDirectory=/opt/astracat-dns/

    Restart=on-failure
    RestartSec=5s
    StandardOutput=journal
    StandardError=journal

    [Install]
    WantedBy=multi-user.target
    ```

    **Важно:** Убедитесь, что строки `ExecStart` и `WorkingDirectory` не содержат комментариев после пути, как это было в примерах отладки. Они должны быть чистыми.

5.  **Перезагрузите Systemd, включите и запустите сервис:**

    ```bash
    sudo systemctl daemon-reload           # Перезагрузить Systemd, чтобы он узнал о новом сервисе
    sudo systemctl enable astracat-dns.service # Включить автозапуск сервиса при загрузке
    sudo systemctl start astracat-dns.service  # Запустить сервис сейчас
    ```

6.  **Проверьте статус сервиса:**

    ```bash
    sudo systemctl status astracat-dns.service
    ```

    Вы должны увидеть `Active: active (running)`.

7.  **Просмотр логов сервиса:**

    ```bash
    sudo journalctl -u astracat-dns.service -f
    ```

    Здесь вы увидите все логи вашего резолвера.
